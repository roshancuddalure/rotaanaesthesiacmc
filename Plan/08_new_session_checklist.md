# New Session Checklist

Use this checklist at the start of every future Codex session for this project.

## Orientation

- [ ] Read `Plan/README.md`.
- [ ] Read `Plan/05_confirmed_decisions.md`.
- [ ] Read `Plan/06_project_operating_principles.md`.
- [ ] Read the current phase in `Plan/07_phased_roadmap.md`.
- [ ] Read latest entries in `Plan/development_log.md`.
- [ ] Check whether unresolved items in `Plan/04_open_questions.md` affect the task.

## Evidence Check

- [ ] Identify which source supports the task:
  - planning docs,
  - user-provided files,
  - existing code,
  - official external documentation if needed.
- [ ] Do not guess silently.
- [ ] If a decision is uncertain, log it as an assumption or open question.

## Work Safety

- [ ] Check Git status before editing.
- [ ] Do not overwrite user changes.
- [ ] Keep edits scoped to the active task.
- [ ] Update tests or add tests when behavior/rules change.

## Documentation

- [ ] Update `Plan/development_log.md` for meaningful changes.
- [ ] Update confirmed decisions if the user answers a policy/product question.
- [ ] Update domain rules if rota/leave logic changes.
- [ ] Update roadmap checklist when phase work is completed.

## Verification

- [ ] Run the relevant tests/checks.
- [ ] Record any checks that could not be run.
- [ ] Summarize changed files and next step.
